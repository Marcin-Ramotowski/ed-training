---
name: review
description: Use this skill when the user asks for a code review, audit, or evaluation of the quality of TypeScript/JavaScript code — either pasted directly or referenced by a file path. Triggers include phrases such as "code review", "check this code", "review the code", "do a review", "code audit", "evaluate this PR", as well as any question about finding bugs or assessing code quality.
---

Perform a detailed code review for the following code or the specified file.

Check:
1. Correctness of business logic
2. Error handling and edge cases
3. Security — especially input validation
4. Readability and naming
5. Compliance with TypeScript strict mode

For each issue, provide: severity level (blocker / warning / suggestion), description of the problem, and a suggested fix.

If the code is good — say so clearly and point out what has been done well.
